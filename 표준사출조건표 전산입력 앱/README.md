# 표준사출조건 전산 모니터링 관리 시스템

태진A&T 사출 SQ 요구사항 — **표준조건 전산화 + 실시간 감시 + 트렌드 평가 + 월간 인쇄(A4 가로)** 시스템.

스택: **Google Sheets (DB) + Google Apps Script (앱)**

## 설치 절차

### 1. 구글시트 생성
1. drive.google.com 에서 새 스프레드시트 만들기 → 이름 예: **`표준사출조건_관리`**
2. 시트 탭 9개 생성: 시트 이름은 TSV 파일명의 `XX_` 접두어를 뺀 것과 동일해야 함
   - `M_품목`, `M_표준조건_UPR`, `M_표준조건_BRKT`, `M_트렌드룰`
   - `D_일일기록`, `D_판정로그`, `M_개정이력`
   - `R_월간데일리`, `R_월실적` (Apps Script가 자동 생성하므로 생략 가능)
3. 각 TSV 파일을 메모장으로 열어 전체 복사 → 해당 시트 `A1`에 붙여넣기
4. **중요**: 한글 깨짐 시 메모장 인코딩 UTF-8 확인

### 2. Apps Script 프로젝트 연결
1. 시트 메뉴: `확장프로그램 → Apps Script`
2. 좌측 파일 추가로 다음 파일들을 생성 후 내용 복붙
   - `Config.gs`, `Web.gs`, `Record.gs`, `Trend.gs`, `Report.gs`
   - `Form.html` (HTML 파일), `Dashboard.html` (HTML 파일)
3. `appsscript.json` 활성화 (프로젝트 설정 → "appsscript.json 매니페스트 파일 표시" 체크) → 내용 붙여넣기
4. `Config.gs` 의 `ALARM_MAIL_TO` 를 실제 수신 메일주소로 변경

### 3. 권한 부여 및 배포
1. `Record.gs` 의 `saveRecord` 임의 실행 → 권한 동의 (스프레드시트/메일/Drive)
2. `배포 → 새 배포 → 유형: 웹앱`
   - 액세스: "도메인 내 모든 사용자" 또는 "나만"
   - 실행: "내 계정으로"
3. 배포 URL을 작업자/관리자에게 공유

### 4. 초기 데이터 채우기
- `M_표준조건_UPR`: UPR CVR 사진의 값으로 이미 채워짐 (목표/상한/하한)
- `M_표준조건_BRKT`: BRKT 사진을 확보하여 동일 방식으로 채움
- `M_품목`의 `사용여부`로 폼 노출 제어

## 사용 방법

### 작업자 입력 (Form)
- 웹앱 URL 접속 → 품목 선택 → 파라메터별 측정값 입력
- 입력 즉시 클라이언트에서 색상 표시 (한계 내 녹색 / 이탈 적색)
- `저장 + 판정` 클릭 → `D_일일기록` + `D_판정로그` 자동 append, NG 발생 시 메일 발송

### 대시보드
- 웹앱 URL + `?page=dashboard`
- KPI 카드: 당월 측정건수, OK/NG, NG율, 주의·이탈 파라메터수
- 최근 이탈/주의 50건 테이블

### 월간 인쇄 (A4 가로)
- 시트의 **메뉴 `표준사출조건`** 에서 클릭:
  - `월간 데일리 생성` → 품목코드/년월 입력 → `R_월간데일리` 시트에 1품목 1개월치 표 생성 (행 31일 × 핵심 30컬럼 + 표준/상한/하한/통계요약)
  - `월실적 요약 생성` → `R_월실적` 시트에 파라메터별 평균/SD/Cp/Cpk/NG건수/등급 산출
  - `현재 리포트 → PDF 저장` → A4 가로 PDF로 Drive에 저장
- 대시보드에서도 동일하게 PDF 출력 가능

### 트렌드 평가 등급
- **안정**: 한계 내, 산포 양호
- **주의**: 연속 7점 한쪽 편향 (CL 기준)
- **경고**: 연속 6점 단조 상승/하강 또는 Cpk < 1
- **이탈**: 1점 ±3σ 초과 또는 한계 이탈 또는 NG율 ≥ 5%

## 데이터 흐름

```
[작업자 폼]
    │
    ├── saveRecord(payload)
    │       ├── D_일일기록 wide append
    │       ├── evaluateRecord_  (M_표준조건과 조인 → OK/NG)
    │       ├── evalTrendGrade_  (M_트렌드룰 + D_판정로그 최근 N건)
    │       ├── D_판정로그 long append
    │       └── NG > 0 → sendAlarm_ (메일)
    │
    └── 대시보드 polling → dashboardKpis

[월간 인쇄]
    generateMonthlyDaily  → R_월간데일리 시트
    generateMonthlySummary → R_월실적 시트
    exportReportPdf → Drive PDF (A4 가로)
```

## 핵심 결정사항 (사진 분석 + Q&A 결과)

| 항목 | 결정 |
|---|---|
| 실린더 7번째 존 | H5 |
| 표준조건 마스터 | 부품별 2종 (UPR CVR / BRKT) |
| 데이터 수집 | 작업자 수동 입력 |
| 트렌드 핵심 파라메터 | 실린더/핫러너 온도, 사출(P/V/S), 보압/쿠션/시간, 중량/금형온도 |
| HOT RUNNER K 박스 | 열전대 타입(K/J) 선택 (온도값 아님) |
| 사출 단수 | 품목별 4~6단 가변 (사용여부=N으로 비활성) |
| V/V GATE | Delay만 정량 관리, Open은 ON/OFF |
| SB1 | 40 = 속도(%), 12.0 = 거리(mm) |
| 인쇄 형식 | A4 가로 / 여백 좁게 / 폰트 7-8pt / 페이지에 맞추기 |
