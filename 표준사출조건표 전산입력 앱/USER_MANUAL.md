## 1. 시작하기

### 1-1. 권장 환경
- **운영체제**: Windows 10 / 11
- **브라우저**: **Chrome 또는 Edge 최신 버전** (Firefox는 일부 기능 제한)
- **인터넷 연결**: 불필요 (모든 동작이 PC 안에서 처리됩니다)

### 1-2. 앱 실행 방법

#### 방법 ① 로컬 서버 실행 (권장 — 자동 저장 가능)
1. **시작 메뉴 → PowerShell** 실행
2. 아래 명령어를 그대로 복사·붙여넣기 → Enter
   ```
   python -m http.server 8000 --directory "C:\Users\USER\Documents\Export\표준사출조건전산관리"
   ```
3. 브라우저 주소창에 **`http://localhost:8000`** 입력 → Enter
4. 작업 종료 시 PowerShell 창에서 **Ctrl + C**

#### 방법 ② 간단 실행 (파일 직접 열기)
- 폴더에서 **`index.html`** 더블클릭 → Chrome / Edge로 열기
- ⚠️ 이 방법은 **폴더 자동저장이 작동하지 않으므로** 매번 다운로드/업로드가 필요합니다

### 1-3. 폴더 연결 (최초 1회)
1. 화면 우측 상단 **`[폴더 선택]`** 클릭
2. 폴더 선택 창에서 **`C:\Users\USER\Documents\Export\표준사출조건전산관리`** 선택
3. *"이 사이트에서 폴더 변경을 허용하시겠습니까?"* → **편집 허용**
4. 좌측 상단 **`📁 표준사출조건전산관리`** (녹색)로 표시되면 연결 완료
5. **`[최신 불러오기]`** 클릭 → 이전 데이터 자동 로드

> 💡 페이지를 새로고침하면 보안 정책상 폴더 권한이 초기화됩니다. 다시 `[폴더 선택]` → `[최신 불러오기]` 순서로 진행하세요.

## 2. 화면 구성 이해하기

### 2-1. 좌측 사이드바 (메뉴)

| 메뉴 | 용도 |
|---|---|
| ✓ **표준사출조건** | 제품별 표준값/UCL/LCL/공차 편집 (관리자) |
| ◧ **사출조건 모니터링** | KPI 카드, 트렌드 차트, 이탈 항목 확인 |
| ⚙ **일일사출조건 입력** | 작업자 측정값 입력, 실시간 신호등 판정 |

### 2-2. 상단 도구바

| 버튼/표시 | 설명 |
|---|---|
| 📂 폴더 미연결 / 📁 폴더명 | 현재 데이터 저장 폴더 상태 |
| **[폴더 선택]** | 데이터 저장 폴더 지정 (Chrome/Edge) |
| **[최신 불러오기]** | 폴더에서 표준조건 + 최근 기록 자동 로드 |
| 🌙 / ☀️ | 다크 / 라이트 테마 전환 |
| ⬤ 빨강 / 노랑 / 초록 | 시스템 표시등 (장식) |

### 2-3. 제품 선택 바

모든 작업 탭의 상단에 표시됩니다.

```
📦 제품  [GARNISH / UPR CVR_수출 ▼]  차종 GARNISH / 품명 UPR CVR_수출  [+ 신규 제품]
```

- **드롭다운**: 등록된 제품 중 1개 선택 → 표준값과 메타 정보가 자동 갱신
- **[+ 신규 제품]**: 새 제품을 등록할 때 사용 ([3-2 참조](#3-2-신규-제품-추가하기))
- **[현재 제품 삭제]** *(표준사출조건 탭에만 표시)*: 현재 선택된 제품 제거

### 2-4. 상태 뱃지 (신호등)

| 뱃지 | 색상 | 의미 |
|---|---|---|
| **OK** | 🟢 녹색 | 한계 이내 (정상) |
| **주의** | 🟡 황색 | 한계 경계 ±5% 이내 (경고) |
| **이탈** | 🔴 적색 (깜빡임) | 한계 초과 (긴급 조치 필요) |
| **-** | ⚪ 회색 | 측정값 미입력 |

## 3. 표준사출조건 등록 (관리자)

> **🔐 관리자 작업** — 라인장/팀장이 매월 또는 제품 변경 시 1회 진행하는 작업입니다.

### 3-1. 제품 선택
1. 좌측 사이드바 → **`표준사출조건`** 클릭
2. 상단 제품 드롭다운에서 편집할 제품 선택

### 3-2. 신규 제품 추가하기
1. 우측 상단 **`[+ 신규 제품]`** 클릭
2. 펼쳐진 폼에 입력:
   - **차종** *(필수)*: 예) GARNISH, PILLAR
   - **품명** *(필수)*: 예) UPR CVR_수출
   - 호기 / TON / 이름 / 원재료명 / CAVITY *(선택, 매번 안 적어도 됨)*
3. **`[추가]`** 클릭 → 새 제품이 등록되고 자동으로 활성화됨
4. 표준값/UCL/LCL을 그 제품에 맞게 수정 후 **`[표준조건 저장]`**

### 3-3. 표준값 / UCL / LCL / 공차 편집
각 그룹별로 파라메터 표가 표시됩니다. **5개의 입력 컬럼**이 있습니다:

| 컬럼 | 의미 | 입력 시 동작 |
|---|---|---|
| **표준값** | 목표 설정값 | UCL/LCL이 현재 ± 만큼 자동 재계산 |
| **상한 UCL** | 허용 최대값 | ± 표시 자동 갱신 |
| **하한 LCL** | 허용 최소값 | ± 표시 자동 갱신 |
| **±** | 공차 (한쪽 편차) | UCL = 표준값 + ±, LCL = 표준값 − ± 자동 재계산 |

#### 입력 팁
- **표준값만 바꾸고 싶을 때**: 표준값 셀만 수정 → 상한/하한 자동 갱신
- **공차만 바꾸고 싶을 때**: ± 셀에 새 공차 입력 → 상한/하한 자동 갱신
- **상한·하한을 비대칭으로 두고 싶을 때**: UCL, LCL을 각각 직접 입력

### 3-4. 그룹별 항목 목록

| 그룹 | 항목 수 | 설명 |
|---|---|---|
| 사출속도 | 6단 | 1단 ~ 6단 (mm/s) |
| 사출압력 | 6단 | 1단 ~ 6단 (RKg) |
| 사출위치(보압전환) | 6단 | 1단 ~ 6단 (mm) |
| 사이클 타임(CT) | 9개 | 충진/사출/쿠션/냉각/보압시간 PH1~PH4/CYCLE TIME |
| 보압압력 | 3단 | 1단 ~ 3단 (bar) |
| 배압 | 3단 | 1단 ~ 3단 (bar) |
| 계량속도 | 3단 | 1단 ~ 3단 (%) |
| 계량완료 | 1개 | 거리 (mm) |
| 실린더온도 | 7개 | 노즐 / H1 ~ H6 (℃) |
| 핫러너 | 79개 | 열전대 타입(K/J) + H1 ~ H78 (℃) |
| 벨브타이머 | 32개 | GATE1 ~ GATE16의 DELAY/OPEN (sec), OPEN은 A/B 타입 선택 |
| 금형온도1 | 2개 | 고정측 / 이동측 (℃) |
| 금형온도2 | 2개 | 고정측 / 이동측 (℃) |
| 칠러온도 | 2개 | 고정측 / 이동측 (℃) |

### 3-5. 저장
- **`[💾 표준조건 저장]`** → 폴더에 `standards.json` 파일이 갱신됨
- 다음에 폴더를 연결하면 이 값이 자동 로드됨
- **`[샘플 5개로 초기화]`** → 모든 제품을 기본 샘플 5개로 되돌림 (⚠️ 변경된 내용 사라짐)

### 3-6. 샘플 등록 제품 (초기 상태)
| 차종 | 품명 | 비고 |
|---|---|---|
| GARNISH | UPR CVR_수출 | 5호기, ABS-XR404, 2 cavity |
| GARNISH | UPR CVR_내수 | 동일 작업자, 내수 사양 |
| GARNISH | BRKT_수출 | PP-T20, 4 cavity |
| PILLAR | UPR CVR_수출 | 김철수, GRAY 색상 |
| PILLAR | BRKT_내수 | PP-T20, 4 cavity |

## 4. 일일사출조건 입력 (작업자)

> **👷 작업자 작업** — 사출 작업 시작 시 또는 정기 점검 시 매일 입력합니다.

### 4-1. 입력 절차 (5분 이내)

#### Step 1. 메뉴 선택
좌측 사이드바 → **`일일사출조건 입력`** 클릭

#### Step 2. 제품 선택
상단 드롭다운에서 작업할 제품 선택
→ **차종 / 품명 / 호기 / TON / 원재료명** 등이 자동 채워짐

#### Step 3. 기본 정보 확인 및 보정
- **년 / 월 / 일** : 오늘 날짜로 자동 입력됨 (수정 가능)
- **이름** : 작업자 본인 이름 (이전 입력 자동 채움)
- **품번 / 색상 / CAVITY** : 필요 시 수정

#### Step 4. 측정값 입력
각 파라메터의 **측정값** 칸에 사출기에서 확인한 값을 그대로 입력합니다.

```
사출속도 1단    표준값 65  상한 75  하한 55   [측정값: 67]   🟢 OK
사출속도 2단    표준값 20  상한 25  하한 15   [측정값: 28]   🔴 이탈
```

- 측정값을 입력하는 순간 **우측 상태 뱃지가 즉시 갱신**됩니다
- **🔴 이탈** 표시 시 → 사출 조건 점검 / 라인장 보고
- **🟡 주의** 표시 시 → 다음 측정에서 추이 확인

> 💡 모든 항목을 다 입력할 필요는 없습니다. **입력한 항목만** 판정됩니다.

#### Step 5. 저장
- 화면 상단 **`[💾 저장]`** 클릭
- 이탈이 있으면 적색 토스트로 알림: *"저장 완료 — ⚠ 이탈 3건"*
- 이탈이 없으면 녹색 토스트: *"저장 완료 — 정상"*

#### Step 6. (옵션) 다음 측정 준비
- **`[🔄 초기화]`** → 측정값만 초기화 (메타 정보는 유지)
- 같은 날 여러 번 측정 시 위 Step 4~5 반복

### 4-2. 입력 화면 구조

```
┌─────────────────────────────────────────────────┐
│ 📦 제품: GARNISH / UPR CVR_수출   [+ 신규 제품]   │
├─────────────────────────────────────────────────┤
│ 기본 정보 [년][월][일][차종][품명]...            │
├─────────────────────────────────────────────────┤
│ 사출속도 (5개 행)                                │
│ 사출압력 (5개 행)                                │
│ 사출위치 (5개 행)                                │
│ ... (15개 그룹)                                  │
└─────────────────────────────────────────────────┘
```

## 5. 사출조건 모니터링

### 5-1. KPI 카드 (당월 누적)

| 카드 | 의미 |
|---|---|
| **측정건수** | 이번 달 저장된 일일기록 횟수 |
| **주의 누적** | 모든 기록에서 발생한 ‘주의’ 건수 합계 |
| **이탈 누적** | 모든 기록에서 발생한 ‘이탈’ 건수 합계 |
| **이탈율(%)** | 이탈 / 총 측정 파라메터 비율 |

### 5-2. 트렌드 차트

1. **그룹 선택** (예: 사출속도)
2. **파라메터 선택** (예: 1단)
3. **기간 선택** (당월 / 최근 30건 / 전체)

→ 꺾은선 그래프 + 가로 점선 3개 (UCL/표준값/LCL) 표시
→ 이탈 점은 **적색**, 주의 점은 **황색**으로 자동 강조

### 5-3. 최근 이탈/주의 목록
최근 20건의 이탈·주의 항목이 시간순으로 표시됩니다. 어떤 파라메터가 얼마나 빈번하게 이탈하는지 확인 가능.

## 6. 데이터 관리

### 6-1. 폴더 안에 생성되는 파일

| 파일명 | 내용 | 갱신 시점 |
|---|---|---|
| `standards.json` | 제품 마스터 + 표준조건/공차 | `[표준조건 저장]` 시 |
| `records-2026-05.json` | 2026년 5월의 모든 일일기록 | `[저장]` 클릭 시 append |
| `records-2026-06.json` | 6월 분 (월 바뀌면 자동 분할) | 자동 생성 |
| `사출조건_2026-05.xlsx` | Excel 변환 결과 | `[📤 Excel]` 클릭 시 |

> JSON 파일은 메모장이나 VSCode로 열어서 직접 백업/검수도 가능합니다.

### 6-2. Excel 내보내기
1. 일일사출조건 입력 탭에서 **`[📤 Excel]`** 클릭
2. 다운로드 폴더에 **`사출조건_YYYY-MM.xlsx`** 파일 생성됨
3. Excel/한컴오피스에서 열어서 인쇄 / 메일 전송 / 보고서 활용

### 6-3. 최신 데이터 불러오기
- 페이지 새로고침 후 또는 다른 PC에서 열 때:
  1. **`[폴더 선택]`** → 작업폴더 선택
  2. **`[최신 불러오기]`** → 표준조건과 최근 1건의 기록 자동 복원

### 6-4. 백업 권장 방법
- 작업폴더 전체를 USB 또는 사내 공유 드라이브에 **주 1회 복사**
- 특히 월말에 `records-YYYY-MM.json` 별도 보관

## 8. 자주 묻는 질문 (FAQ)

### Q1. `[폴더 선택]` 버튼을 눌렀는데 폴더 창이 안 떠요
**A.** Chrome 또는 Edge **최신 버전**에서만 작동합니다. Internet Explorer / Firefox에서는 지원되지 않습니다. 또한 **`file://`** 경로(파일 더블클릭)로 열었을 때도 작동하지 않으므로 [1-2 방법 ①](#1-2-앱-실행-방법) 으로 로컬 서버를 실행하세요.

### Q2. 새로고침 후 데이터가 사라졌어요
**A.** 데이터는 사라지지 않았습니다. 폴더의 JSON 파일에 안전하게 저장되어 있습니다. 새로고침 후에는:
1. **`[폴더 선택]`** → 작업폴더 다시 선택
2. **`[최신 불러오기]`** → 표준조건 + 최근 기록 복원

### Q3. 측정값을 입력했는데 신호등이 회색이에요
**A.** 해당 파라메터의 표준값이 등록되지 않았을 수 있습니다. **`표준사출조건`** 탭에서 해당 제품의 그 항목에 표준값/UCL/LCL을 입력하세요.

### Q4. 이탈인데 신호등이 OK로 나와요
**A.** 다음 중 하나를 확인하세요:
- UCL/LCL이 너무 넓게 설정되어 있지 않은가
- 표준조건을 저장(💾)하지 않은 채 측정값을 입력했는가
→ 표준조건 탭에서 값 확인 → **`[표준조건 저장]`** → 일일입력 탭으로 가서 다시 측정값 입력

### Q5. 제품을 잘못 선택했어요. 저장 후 변경 가능한가요?
**A.** 저장된 기록은 측정 시점의 제품ID로 보존됩니다. 잘못 입력한 경우:
- `records-YYYY-MM.json` 파일을 메모장으로 열어 해당 항목 삭제
- 또는 새로 올바른 제품 선택 후 다시 입력 (이전 기록과 함께 누적됨)

### Q6. Excel 내보내기 결과가 이상해요
**A.** Excel을 한국어 Microsoft Office 또는 한컴오피스에서 여세요. WPS Office는 일부 헤더 병합이 안 보일 수 있습니다.

### Q7. 핫러너 H78까지 다 입력해야 하나요?
**A.** **사용하는 존만 입력**하세요. 미사용 존은 비워두면 판정 대상에서 제외됩니다.

### Q8. 같은 날 여러 번 측정한 기록을 모두 보관할 수 있나요?
**A.** 네, 가능합니다. 측정마다 별도 기록으로 저장되며, 모니터링 탭의 트렌드 차트는 시간순으로 모두 표시합니다.

### Q9. 다른 PC에서도 같은 데이터를 사용할 수 있나요?
**A.** 작업폴더 전체를 USB 또는 네트워크 공유로 복사하세요. 새 PC에서 같은 폴더를 `[폴더 선택]`으로 연결하면 동일하게 작동합니다.

### Q10. 항목을 추가하거나 그룹명을 바꾸고 싶어요
**A.** 시스템 관리자/개발 담당자에게 요청하세요. 개발자가 `js/schema.js` 파일에서 파라메터 정의를 추가/수정합니다.
